# Skill: ux-carbon-critique

    ## Intent
    Review UI against IBM Carbon guidance, accessibility expectations, and layout/typography consistency.

    ## Inputs
    - Screenshots or Storybook link
- Relevant user journeys
- Carbon component list in use

    ## Outputs
    - UX critique (findings + severity)
- Suggested improvements (Carbon-aligned)
- A11y notes and tests to run

    ## Process
    1. Validate layout hierarchy and spacing.
2. Check Carbon component usage and theming consistency.
3. Assess accessibility (contrast, focus, labels, keyboard).
4. Suggest concrete UI changes and acceptance criteria.

    ## Quality bar
    - Actionable, prioritised findings.
- Each finding has a suggested fix and validation step.

    ## Escalation triggers
    - Design decision conflicts with product requirements
- Requires new design system tokens or patterns

    ## Tooling notes
    Follow the calling agent's tool allow/deny list.
